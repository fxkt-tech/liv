# FFVMix 模板编译器

## Goal

建立可持久化 FFVMix 模板及其一次性编译流程，把本地文件、字幕和时长策略转换为经过完整校验的不可变 `CompiledTemplate`。

## Requirements

- 模板必须包含画布、背景、默认配置、有序必选槽位、连接转场池、BGM 池、有序全局图层和内置约束配置。
- 槽位、候选、转场、BGM 和图层 ID 由构造器生成并持久化；JSON 缺失 ID 时失败。
- 相对路径只按显式 base directory 解析。
- 视频候选支持完整文件或源区间、权重、固定/可变时长策略、画面适配和原声音量。
- 编译必须去重探测本地素材，固定使用第一路视频/音频流；视频流必需，音频流可缺失。
- FFprobe 时长必须以足够精度转换为微秒，不能继续依赖 `float32`。
- 编译必须验证源区间、速度范围、转场引用、图层锚点和内置约束配置。
- SRT/ASS 必须在编译时转换为结构化字幕。
- 默认文件指纹为 size + mtime，可选 SHA-256。
- 所有错误必须聚合返回；不能静默跳过候选。
- `CompiledTemplate` 不可变，包含后续生成所需的全部元数据和预计算适配信息。

## Acceptance Criteria

- [ ] 模板 JSON round-trip 保留所有 ID、顺序和策略。
- [ ] 同一文件被多个候选引用时只探测一次。
- [ ] 相对路径不受进程当前目录影响。
- [ ] 文件缺失、无视频流、区间越界和无效引用可一次返回多个错误。
- [ ] 无音频流的视频能成功编译并标记为静音。
- [ ] speed/trim/loop/freeze/reject 方案能被确定地预计算。
- [ ] SRT/ASS 与结构化字幕最终得到相同的规范化 cue 类型。
- [ ] 编译结束后生成阶段不需要再次 FFprobe。
- [ ] 单元测试使用 fake prober；真实 FFprobe 集成测试使用仓库 fixture 或动态生成素材。

## Out of Scope

- 组合枚举、历史约束和 `Next`。
- 构建最终绝对时间线 Project。
- 远端素材。
- FFmpeg 渲染。
