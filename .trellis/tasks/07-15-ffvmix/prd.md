# FFVMix 混剪模块

## Goal

提供一个可持久化的混剪模板系统：模板由必选的顺序视频槽位、转场、背景音乐和确定性的全局图层组成；调用方可以按需、可复现地生成多个互不重复且满足约束的 `ffcut.Project` 合成协议。

第一阶段只生成协议，不执行 FFmpeg 渲染。

## Background

- 现有 `ffcut/fusion` 同时承担协议构建和 FFmpeg 执行，且无法完整表达本需求。
- 现有协议没有兼容要求，可以直接建立新的 `ffcut.Project v2`。
- 素材暂时只来自本地文件。

## Requirements

### R1. 模板与槽位

- 模板包含有序槽位；槽位表示时间线上的固定片段位置。
- 每个槽位必须至少有一个有效视频候选，每条结果必须为每个槽位选择一个候选。
- 槽位和候选 ID 由 `ffvmix` 构造器生成并持久化；外部 JSON 缺少 ID 时校验失败。
- 候选可以引用完整本地文件或明确的源区间。
- 候选、转场和 BGM 支持权重；权重只影响生成顺序。

### R2. 时长适配

- 槽位未指定时长时，结果片段使用候选源区间的原始时长。
- 固定时长槽位对超长素材支持 `speed_up / trim / reject`。
- 固定时长槽位对不足素材支持 `slow_down / loop / freeze / reject`。
- 变速受模板默认值和槽位覆盖值约束；越界时拒绝候选，不隐式降级。
- 裁剪支持 `start / center / end / random`；随机位置由生成种子决定，同一“候选 × 槽位”不产生无限变体。

### R3. 视频、转场与音频

- 模板配置输出画布宽高和可选 FPS，默认 30 FPS。
- 素材适配支持 `cover / contain / stretch`，默认 `cover`。
- 背景支持纯色、图片和当前槽位视频的模糊填充，默认黑色。
- 相邻槽位的连接可以配置多个转场候选，`cut` 也是候选。
- 固定槽位时长包含转场重叠；总时长扣除转场重叠时间。
- 转场时长不可用时跳过该组合，不自动缩短；转场默认同时交叉淡化相邻视频原声。
- 视频固定使用第一路视频流和第一路音频流；没有音频流时按静音处理。
- 原声和 BGM 分别配置音量，槽位可覆盖原声音量。
- BGM 支持候选池、起始偏移、截断、循环和淡入淡出，但不做变速。

### R4. 全局图层

- 支持结构化字幕、本地 SRT/ASS、水印和贴图。
- SRT/ASS 在模板编译阶段解析为结构化字幕。
- 图层不参与混剪组合，按模板声明顺序从下到上绘制。
- 时间锚点支持 `full_output`、槽位内偏移和成片绝对时间。
- 空间定位支持锚点，默认 `top_left`；位置和尺寸支持像素与画布百分比。
- 百分比取值为 `0–100`；横向值相对画布宽度，纵向值相对画布高度。

### R5. 素材编译

- 相对路径基于显式模板目录解析，不依赖进程工作目录。
- 创建生成器前一次性探测所有唯一素材，并集中返回全部校验错误。
- 默认记录文件大小和修改时间，可选 SHA-256 严格指纹。
- 编译结果不可变；惰性生成阶段不得再次访问素材元数据。

### R6. 惰性生成

- 不预设输出数量；调用方通过串行 `Next(ctx)` 按需消费。
- 同一个生成器不支持并发调用，也不支持第一版跨进程 checkpoint。
- 种子可省略；省略时由 `ffvmix` 生成并暴露。
- 相同模板、素材指纹、约束和种子必须生成相同顺序。
- 完全相同的槽位、转场和 BGM 组合只能产生一次。
- 单次搜索有预算；超出预算与组合耗尽必须返回不同状态。

### R7. 约束

- 约束是无副作用的插件，只读取候选摘要和已接受历史摘要。
- 约束失败返回拒绝原因；只有生成器能提交结果并更新历史。
- 使用按种子顺序的贪心筛选，不回溯，也不保证全局最大结果集合。
- 模板持久化内置约束配置；自定义 Go 约束在运行时注入。
- 内置 `MaxSimilarity` 按相同槽位同素材的重叠视频时长计算，不包含固定图层和 BGM。
- 内置视频素材复用约束默认按规范化文件路径统计；同一文件的不同片段共享额度。
- 视频素材与 BGM 使用独立的复用约束。

### R8. 输出协议与溯源

- `Next` 返回类型化的 `ffcut.Project v2`，JSON 序列化是独立操作。
- Go 内部时间使用 `time.Duration`，协议时间序列化为 `int64` 微秒。
- Project 记录模板摘要、随机种子、组合指纹、槽位选择、素材指纹和约束配置摘要。
- 当前 `ffcut` JSON 协议不要求兼容。

## Child Tasks

1. `07-15-ffcut-project-v2`：建立共享时间线协议。
2. `07-15-ffvmix-template-compiler`：建立模板、素材探测和不可变编译结果；依赖 1。
3. `07-15-ffvmix-generator`：建立惰性组合、约束和 Project 生成；依赖 1、2。

## Acceptance Criteria

- [ ] AC1：模板可以保存并重新加载，缺失内部 ID 或包含无效引用时返回可定位的校验错误。
- [ ] AC2：每个有效结果恰好选择每个槽位的一个候选，并正确应用固定/可变时长策略。
- [ ] AC3：转场、原声、BGM、背景和全局图层能完整表示在类型化 Project 中。
- [ ] AC4：本地素材只探测一次；无视频流、区间越界等错误在编译阶段集中返回。
- [ ] AC5：相同输入和种子产生相同结果顺序，结果中不存在完全重复组合。
- [ ] AC6：两个内置约束通过自动化测试，自定义纯约束可以组合使用。
- [ ] AC7：`Yielded / Exhausted / BudgetExceeded` 三种生成状态可稳定区分。
- [ ] AC8：Project JSON round-trip 不丢失语义，时间使用微秒且通过校验。
- [ ] AC9：所有新单元测试不依赖开发者桌面文件；集成测试素材可在测试中生成或使用仓库 fixture。
- [ ] AC10：第一阶段不新增 FFmpeg 渲染路径，`ffvmix` 不依赖 `ffmpeg/filter`。

## Out of Scope

- 远端素材和远端元数据解析。
- 实际生成视频文件的 `ffcut/renderer`。
- 可选槽位。
- 图层候选参与组合。
- 全局最优约束求解。
- 同一生成器并发消费和跨进程恢复。
